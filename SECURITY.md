# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Email:** [kogunlowo@gmail.com](mailto:kogunlowo@gmail.com)

Please include:
- A description of the vulnerability.
- Steps to reproduce the issue.
- Any potential impact or severity assessment.

We will acknowledge receipt within 48 hours and aim to provide a resolution timeline within 5 business days.

## Security Best Practices

When using this module, please follow these security best practices:

- **Pin module versions** to avoid unexpected changes.
- **Use least-privilege IAM roles** for all service accounts and users.
- **Enable audit logging** to track access and changes.
- **Store state files securely** using encrypted remote backends with access controls.
- **Never commit secrets** such as credentials, API keys, or service account keys to version control.
- **Review plans carefully** before applying changes to production environments.
